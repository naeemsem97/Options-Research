## How Can I Contribute?

### Reporting Bugs or Issues
- Use the GitHub Issues tab
- Describe the issue clearly
- Include steps to reproduce if applicable
- Share any error messages or unexpected behavior

### Suggesting Enhancements
- Open an issue with your suggestion
- Explain why this enhancement would be useful
- Provide examples if possible

### Submitting Code
1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test your code (if applicable)
5. Commit with a clear message (`git commit -m 'Add some feature'`)
6. Push to your branch (`git push origin feature/your-feature-name`)
7. Open a Pull Request

### Sharing Ideas
- Have an interesting volatility strategy idea?
- Found a cool paper or article on options trading?
- Want to discuss implementation approaches?

Just open an issue or discussion! No idea is too small or too basic.

## Code Style
- Keep it simple and readable
- Add comments to explain complex logic
- Follow PEP 8 guidelines for Python code (but don't stress too much!)
- Include docstrings for functions and classes

## Strategy Contributions
When contributing a new strategy:
- Place it in the appropriate directory (volatility, spreads, or directional)
- Include a clear description at the top of the file
- Document entry/exit conditions
- Add comments explaining your reasoning
- Consider including backtest results or observations

## Questions?
Don't hesitate to ask! Open an issue or reach out. We're all learning here.
